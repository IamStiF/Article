Autores

Problema

Los autores del artículo reconocen y plantean soluciones frente a los ataques CSS de origen cruzado. Estos ataques son básicamente una técnica de ataque en la que un atacante puede inyectar código CSS malicioso en una página web vulnerable, que luego se ejecuta en el navegador de la víctima. Esto mediante entradas del usuario que no son filtradas correctamente y gracias problemas con el manejo de errores sintácticos de CSS, se permiten el “linkeo” de hojas de estilo CSS maliciosas que se cargan al servidor web y afectan al contenido de la web. El objetivo del ataque es modificar la apariencia visual de la página web para engañar a la víctima y hacer que haga clic en enlaces maliciosos o proporcione información confidencial solicitandoles esa información en formularios con una apariencia similar a la página web que se intentó atacar. Todo esto sin el uso de Javascript, únicamente explotando vulnerabilidades con CSS. En este artículo los autores plantean que para prevenir este tipo de ataques, es importante que los desarrolladores web filtren adecuadamente la entrada del usuario y validen todos los datos para asegurar de que no contengan sentencias maliciosas, antes de enviarlas al servidor. También se recomienda el uso de herramientas de seguridad en la capa de aplicación, como firewalls de aplicaciones web, y tener una política de seguridad de contenido (CSP), la cual es una capa adicional de seguridad que permite a los desarrolladores web controlar qué recursos se pueden cargar en un sitio web, incluidas las hojas de estilo.

Otras Soluciones

Newlines

El estándar CSS no permite que las cadenas de texto y las URLs contengan saltos de línea. La mayoría de los navegadores hacen cumplir esta regla para prevenir ataques CSS de origen cruzado. Para defenderse contra este tipo de ataques, se pueden insertar saltos de línea antes y después de los posibles puntos de inyección.

HTML Encoding

Es una técnica utilizada para proteger las aplicaciones web contra ataques XSS que consisten en la inyección de código malicioso en una página web. Esta técnica implica la sustitución de ciertos caracteres especiales, como las comillas y los corchetes, con códigos HTML conocidos como "entidades HTML". Al hacerlo, los caracteres especiales se convierten en texto que el navegador web interpreta como texto plano, en lugar de como código ejecutable. Además, es importante asegurarse de que la página web tenga una declaración de conjunto de caracteres adecuada, para evitar que un atacante pueda descifrar la codificación HTML sustituyendo la página a UTF-7.

Avoid Ambient Authentication

El navegador transmite información de autenticación con cualquier solicitud al sitio de destino. El esquema de autenticación de clave web evita el uso de esta información ambiental y bloquea varios tipos de ataques. Sin embargo, si una URL con credenciales se vuelve visible para el usuario víctima, podría ser engañado para revelarlas, por lo que se debe evaluar si este riesgo es aceptable.

Experimento

Para la demostración de la hipótesis correspondiente al problema planteado, se usa un modelo de amenazas en el cual se exponen tres principales aspectos. Estos aspectos son la exposición de las habilidades del atacante, el comportamiento que se espera y se cree que toma el atacante para la realización del ataque y el comportamiento de la víctima en el mismo. Habilidades del atacante. Primeramente, se debe mencionar que este corresponde a una persona maliciosa que cuenta con un dominio y opera en un servidor web. Su objetivo es robar información confidencial de otro sitio web que debería de ser únicamente para la víctima y no el atacante. Comportamiento esperado por parte del atacante. Esta persona puede manipular tráfico de red, pero únicamente desde sus propios servidores. Además, no puede escuchar o modificar el tráfico de red de otros usuarios o sitios web. Este puede generar paquetes falsos para intentar acceder al lado del usuario. Sin embargo, no es posible que instale en la computadora de la víctima. Comportamiento de la víctima. Lo más probable es que el atacante envíe publicidad engañosa para hacer a la víctima caer y acceder en sitios web maliciosos para de esta forma, realizar el ataque enviando información a los servidores. Es importante mencionar que, el CSS cuando hay errores no ocurre un problema como tal para poder llevar a cabo las demás instrucciones. En el momento en que se encuentra un error, simplemente se salta a la línea siguiente y se continúan ejecutando las instrucciones. Es por esto, que muchos ataques se intentan hacer por medio de CSS.
